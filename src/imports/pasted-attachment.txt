# Client-Side Redirects for Old URLs

## Instruction for Figma AI

Google has indexed old URLs that no longer work. We need to redirect them to their correct new URLs. Since we cannot do server-side 301 redirects (Cloudflare Worker doesn't intercept requests on earnbase.finance), implement client-side redirects in the app's router.

## How It Works

On page load (or in the router), check if the current URL path matches any entry in the redirect map below. If it does, immediately redirect to the new URL using `window.location.replace()`. This must happen BEFORE the page renders any content — no flash of "No products available".

## Important Notes

1. This redirect check MUST run before the router renders any page. If the user sees "No products available" for even a split second before redirecting, it means the check runs too late.

2. Use `window.location.replace()` not `window.location.href`. Replace does not create a back-button entry — the user won't get stuck in a redirect loop when pressing back.

3. This is a static map — it only covers URLs that Google has already indexed. It does not need to grow unless we discover more old URLs in Google's index.
